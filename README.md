# FocusOS 🎯

> **"Control what deserves your attention."**

FocusOS is a privacy-first personal operating system that acts as an intelligent layer on top of your notifications and emails. It turns scattered digital conversations into structured, searchable memory—fully on-device.

---

## ❗ The Problem
We regularly lose critical information across apps like WhatsApp, Instagram, Slack, and Gmail. This leads to missed deadlines, forgotten commitments, and lost context. Meanwhile, user concerns regarding privacy, surveillance, and data misuse are growing rapidly. 

## 💡 The Solution
FocusOS extracts only meaningful signals from your notifications, processing everything locally using a **Zero-Trust Privacy Architecture**. You retain complete control over your data.

---

## 🔐 Privacy Principles (Non-Negotiable)

1. **On-Device First**: All data processing and filtering happens locally on your hardware. **No raw message data ever leaves your device.**
2. **Minimal Data Collection**: Only stores the filtered notification text, sender, unassociated tags, and the timestamp.
3. **Automatic Sensitive Filtering**: Native interceptors automatically ignore OTPs, passwords, banking information, and verification codes instantly.
4. **End-to-End Local Encryption**: All stored memory chunks are locked using mathematically secure `AES-256` keys generated by the hardware keystore.
5. **Transparency & Control**: The built-in Privacy Dashboard lets you view exactly what is tracked, and instantly wipe everything with a single tap.

---

## 🧩 Core Features (MVP)

- 📜 **Unified Memory Feed**: A clean, minimal timeline of captured *actionable* messages.
- 🔍 **Smart Search**: Fast, local indexing to instantly recall commitments, times, and deadlines.
- 🧠 **Importance Detection**: Rule-based engine hunting for deadlines ("tomorrow", "5pm") and commitments ("I will", "\$").
- 🎛️ **App Filters**: Granular toggles to track or ignore specific platforms.
- 🛡️ **Privacy Dashboard**: Monitor your local memory footprint and securely delete your data.

---

## ⚙️ Technical Architecture

- **Frontend Environment**: Flutter (Material / Minimalist Dark UI)
- **Local Embedded Storage**: `hive_flutter` alongside `flutter_secure_storage` for AES-encrypted persistence.
- **Native Subsystem (Android)**: Custom Kotlin `FocusNotificationListenerService` capturing background events and dropping unneeded configurations before ever reaching the Flutter VM layer.
- **Communication Bridge**: Synchronous `MethodChannel` mapping native Android Broadcasts dynamically.

---

## 🚀 Future Scope

This foundational architecture ensures that future expansions can run entirely in the background:

## 🛠️ Testing & Compilation

This application relies natively on Android's `NotificationListenerService`. Thus, to fully utilize memory feed tracking, it **must run on an Android device or API-equipped emulator**.

1. **Clone the Source:**
   ```bash
   git clone https://github.com/mridulsharma-se/focus_os.git
   cd focus_os
   ```
2. **Install Flutter Dependencies:**
   ```bash
   flutter pub get
   ```
3. **Execute:**
   ```bash
   flutter run
   ```
4. **Grant Core Permissions:** Keep an eye out for initial prompts, or manually navigate to `Settings -> Apps -> Special App Access -> Notification Access` and toggle on **FocusOS**.
